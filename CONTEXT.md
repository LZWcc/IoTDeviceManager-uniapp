# IoT 设备管理上下文

本上下文描述 IoT 设备管理与监控系统中的业务语言，用于统一前端、后端、数据库和底层设备上报之间的概念边界。

## Language

**设备**:
系统管理和监控的物理 IoT 终端。
_Avoid_: 设备实例、设备记录

**设备编号**:
设备在业务界面和底层上报中共同使用的外部编号，不等于数据库主键。
_Avoid_: 设备 ID、设备编码、电车编号 id

**传感器数据**:
设备上报的环境、状态或测量类数据。
_Avoid_: sensor 数据、实时数据

**行为数据**:
设备上报的行为或动作状态类数据，不表示用户行为。
_Avoid_: behavior 数据、用户行为数据

**数据字段映射**:
把底层上报字段名、数据库存储字段名和界面展示字段名对应起来的配置。
_Avoid_: mapper、字段表

**上报字段名**:
底层设备消息里使用的字段名。
_Avoid_: p_name

**存储字段名**:
数据库数据表里保存上报值的字段名。
_Avoid_: db_name

**展示字段名**:
界面中展示给用户看的字段名称。
_Avoid_: f_name

**实时数据视图**:
面向当前监控场景展示最近上报数据的查询视图。
_Avoid_: 实时数据类型、实时数据表

**历史数据视图**:
按时间范围查询过往上报数据的查询视图。
_Avoid_: 历史数据类型、历史数据表

**指令模板**:
定义可配置指令项及其层级、可选值和展示方式的模板。
_Avoid_: direct config、配置表

**指令配置值**:
在全局范围或某个设备编号范围内保存的指令项取值。
_Avoid_: 指令实例、用户配置

**全局配置**:
作用于所有设备的指令配置值集合。
_Avoid_: GLOBAL 设备、全局设备

**设备配置**:
作用于单个设备编号的指令配置值集合。
_Avoid_: 设备实例配置

**错误信息**:
设备上报的异常记录，可按类型区分为告警或错误。
_Avoid_: 异常信息、事件信息

**告警**:
错误信息中表示风险或异常状态提醒的类型。
_Avoid_: 警告

**错误**:
错误信息中表示故障或错误状态的类型。
_Avoid_: 故障

**数据来源类型**:
描述一条上报数据记录来源或保存方式的标签。
_Avoid_: 是否在线数据、在线状态

## Relationships

- 一个 **设备** 拥有一个用于上报、筛选和下发指令的 **设备编号**。
- 一个 **设备编号** 可以关联多条 **传感器数据**、**行为数据**、错误信息和设备级指令配置。
- **传感器数据** 和 **行为数据** 是两类不同的数据上报通道，但都以 **设备编号** 归属到设备。
- 一条 **数据字段映射** 连接一个 **上报字段名**、一个 **存储字段名** 和一个 **展示字段名**。
- **实时数据视图** 和 **历史数据视图** 都查询 **传感器数据** 或 **行为数据**，不是独立的数据类别。
- 一个 **指令模板** 可以产生多份 **指令配置值**：一份属于 **全局配置**，也可以分别属于多个 **设备配置**。
- **全局配置** 不属于任何 **设备编号**；代码中的 `GLOBAL` 只是全局作用范围的哨兵值。
- 一条 **错误信息** 属于一个 **设备编号**，并且类型可以是 **告警** 或 **错误**。
- **数据来源类型** 描述的是上报记录本身，不等同于设备当前是否在线。

## Example dialogue

> **Dev:** “一个设备可以没有设备编号吗？”
> **Domain expert:** “业务上用于上报、筛选和下发指令的设备需要设备编号；数据库主键只是内部记录编号。”
>
> **Dev:** “历史数据筛选里的设备编号和设备表的 id 是同一个东西吗？”
> **Domain expert:** “不是。设备编号是底层上报和界面筛选用的编号，数据库 id 只是内部主键。”
>
> **Dev:** “行为数据是用户点击页面产生的行为吗？”
> **Domain expert:** “不是。行为数据是设备自己上报的行为或动作状态，和传感器数据一样都归属到设备编号。”
>
> **Dev:** “底层字段 `t1` 为什么在页面上显示成水温？”
> **Domain expert:** “这是数据字段映射决定的：`t1` 是上报字段名，保存到某个存储字段名，再用展示字段名显示给用户。”
>
> **Dev:** “实时数据和历史数据要不要分成两套领域模型？”
> **Domain expert:** “不用。它们是同一类上报数据的不同查询视图：实时数据视图看最近数据，历史数据视图按时间范围看过往数据。”
>
> **Dev:** “`GLOBAL` 是一个特殊设备编号吗？”
> **Domain expert:** “不是。`GLOBAL` 只是实现里表示全局配置作用范围的哨兵值，全局配置不属于某个设备编号。”
>
> **Dev:** “错误信息页面里显示的告警算不算错误信息？”
> **Domain expert:** “算。错误信息是上位词，告警和错误是两种错误信息类型。”
>
> **Dev:** “数据记录里的‘是否在线数据’能代表设备现在在线吗？”
> **Domain expert:** “不能。它应理解为数据来源类型；设备当前在线状态要看心跳。”

## Flagged ambiguities

- `d_no`、设备编码、设备编号、设备表 `number` 曾混用；已统一：领域语言使用 **设备编号**。
- `showDeviceFeatures` 这个实现名容易被理解成设备功能总开关；已澄清：它控制页面中是否显示设备编号。
- 数据表字段 `online` 和页面文案“是否在线数据”容易被理解成设备在线状态；已澄清：领域语言使用 **数据来源类型**，设备当前在线状态由心跳判断。
